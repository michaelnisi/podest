  • message(describing:)

    Returns an error message describing error or nil if the error can be ignored.

    Here’s the reasoning behind these messages.

    • A message is available and should be shown to the user.
    • No message is required, returning nil, the error can be ignored.

    • Unknown, an unknown error message is returned and might be displayed, but actually, handling is undefined.
