  • update(considering:completionHandler:)

    Updates the user’s queue, requesting downloads included, encapsulated into a stand-alone operation with a callback block, designed for background fetching.

    Use this for background fetching, when this completion handler executes, we are ready for a new snapshot of the UI. This method is allowed 30 seconds of wall-clock time before getting terminated with 0x8badf00d.

    QA

    Declaration

    Swift

    func update(considering error: Error?, completionHandler: @escaping ((_ newData: Bool, _ error: Error?) -> Void))

    Parameters

    error

    An error if something went wrong.
    completionHandler

    The block to execute when the queue has been updated AND the view has been refreshed.
    newData

    true if new data has been received.
    error

    An error if something went wrong.
  • reload(completionBlock:)

    Reloads queue, missing items might get fetched remotely, but saving time the queue doesn’t get updated.

    Use update(completionHandler:) to update, which includes reloading.
